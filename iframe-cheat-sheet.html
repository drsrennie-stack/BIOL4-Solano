Link vs Embed Decision

Before reaching for an iframe, ask whether a direct link would serve students better. Here is the quick decision matrix:

Canvas (SCC course)

Canvas wraps content in its own UI chrome. Embedding adds a second wrapper around your tool, breaking interactivity and making the layout cramped.

Direct link, opens in new tab

Kajabi (business sites)

Kajabi pages are designed to host embedded tools. The iframe pattern with the height-sender script gives a seamless feel, with no second scrollbar.

Embed with height-sender

Sharing the URL directly

For peer review, accreditation evidence, or anywhere the tool needs to live independently. The GitHub Pages URL is the artifact.

Just send the link

Kajabi Embed Code

In Kajabi, add a Custom Code block on the page or post where you want the tool to appear. Paste the code below and replace the id and src values.

Kajabi custom code block 
<iframe
  id="my-tool"
  src="https://drsrennie-stack.github.io/REPO-NAME/FILE.html"
  style="width: 100%; border: 0; min-height: 400px;"
  loading="lazy"
  title="Tool description for screen readers">
</iframe>

<script>
window.addEventListener('message', function(e) {
  if (e.data && e.data.type === 'iframeHeight' && e.data.id === 'my-tool') {
    var iframe = document.getElementById('my-tool');
    if (iframe) iframe.style.height = e.data.height + 'px';
  }
});
</script>

The two pieces

  • The iframe tag loads your tool. The id must match what you target in the script. loading="lazy" defers loading until the iframe is near the viewport, which speeds up the page.
  • The receiver script listens for height messages from the embedded tool and resizes the iframe to fit. Without this, you get a fixed-height iframe with internal scrollbars.
Multiple tools on one page? Give each iframe a different id (like tool-1, tool-2) and duplicate the receiver script's id check for each one. Or use a single script that checks all expected ids.

Canvas Embed Code (if you must)

If you need to embed a tool inside a Canvas page rather than linking out, Canvas provides an iframe option through the rich content editor. The receiver script will not run inside Canvas pages, so the height-sender will not auto-resize. Set a sensible min-height instead.

Canvas rich content editor, HTML view 
<iframe
  src="https://drsrennie-stack.github.io/REPO-NAME/FILE.html"
  style="width: 100%; height: 800px; border: 0;"
  title="Tool description for screen readers"
  loading="lazy">
</iframe>

To paste this in Canvas: open the rich content editor, click the HTML view button (looks like </>), and paste the code in. Adjust the height value to fit your tool's typical content size.

Recommendation: For most Canvas use cases, link directly to the GitHub Pages URL instead. A Canvas page item can be set to "External URL" and Canvas will open it in a new tab. Cleaner UX, full viewport, no iframe quirks.

Height-Sender Pattern

This is your standing rule for all GitHub Pages tools. The pattern has two halves: the sender lives inside your tool's HTML file, the receiver lives on the parent page (Kajabi).

Sender, baked into every tool's HTML before </body>

Sender (in your tool's HTML) 
<script>
(function() {
  function sendHeight() {
    try {
      var height = document.documentElement.scrollHeight;
      var id = window.frameElement ? window.frameElement.id : '';
      window.parent.postMessage({ type: 'iframeHeight', id: id, height: height }, '*');
    } catch (e) { /* ignore cross-origin errors when not embedded */ }
  }
  window.addEventListener('load', sendHeight);
  window.addEventListener('resize', sendHeight);
  if ('ResizeObserver' in window) {
    new ResizeObserver(sendHeight).observe(document.body);
  }
  document.querySelectorAll('details').forEach(function(d) {
    d.addEventListener('toggle', sendHeight);
  });
})();
</script>

What this script does

  • Sends the document height to the parent page on load, on resize, and whenever any element resizes (via ResizeObserver).
  • Also sends on toggle of any <details> element, so collapsibles auto-resize the iframe.
  • Uses window.frameElement.id to identify which iframe sent the message, so multiple iframes on one page can coexist.
  • Wrapped in try/catch so the script does not throw errors when the page loads outside an iframe.

Receiver, on the Kajabi page (already shown in the Kajabi section above)

The receiver script reads the message and updates the iframe height. The id in the message must match the iframe's id attribute.

Link Target Rules

When your tool is loaded inside an iframe, links inside the tool can behave in unexpected ways. Use these target rules to keep navigation predictable.

Link type Target attribute Why
Internal anchor (#section) None needed Anchors scroll within the iframe, which is what you want
Same-domain page (your other tools) target="_top" Breaks out of the iframe so the new page takes the full viewport
External URL (someone else's site) target="_blank" rel="noopener" Opens in a new tab, rel="noopener" blocks the destination from accessing your window
mailto: or tel: target="_top" Lets the device handle the link (open email or dialer)
Examples 
<!-- Anchor scroll, no target -->
<a href="#schedule">Jump to schedule</a>

<!-- Internal page, breaks out of iframe -->
<a href="/BIOL4-Solano/syllabus.html" target="_top">Course syllabus</a>

<!-- External link, new tab -->
<a href="https://openstax.org" target="_blank" rel="noopener">OpenStax</a>

<!-- Email or phone, breaks out -->
<a href="mailto:sharilyn.rennie@solano.edu" target="_top">Email me</a>

Problems & Fixes

Iframe shows a scrollbar instead of expanding

The receiver script is missing or has a mismatched id. Confirm:

  • The receiver script is on the parent (Kajabi) page
  • The id on the iframe matches the id check in the script
  • The tool's HTML actually has the sender script before </body>

Internal link inside the tool stays trapped in the iframe

Add target="_top" to the link. This is the most common iframe gotcha.

Iframe is blocked entirely (X-Frame-Options error)

Some sites send headers that prevent embedding. GitHub Pages does not, so your tools will always embed fine. If you are trying to embed someone else's site and getting blocked, that site is not embeddable. There is no workaround on your end.

Iframe is too tall on mobile

The sender script handles this automatically. If it is not working on mobile, check that the tool itself is responsive (uses relative widths, viewport meta tag set correctly). The iframe can only fit the content it is given.

Tool loads but features (like fetch, or audio) do not work

Some browser features are restricted in iframes by default. Add the allow attribute:

Granting iframe permissions 
<iframe
  src="..."
  allow="autoplay; microphone; camera; clipboard-write">
</iframe>

Only request what you actually need. Most of your tools (study guides, generators, viewers) need none of these.

Updates to the tool are not showing up

GitHub Pages caches aggressively. Two fixes:

  • Hard refresh in the browser (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows)
  • Append a version query string to the iframe src: file.html?v=2. Update the version each time you push a meaningful change.

Pre-flight Checklist

Before pushing a tool live or embedding it, run through this list:

Inside the tool's HTML

  • Sender script is in place before </body>
  • All same-domain links have target="_top"
  • All external links have target="_blank" rel="noopener"
  • Mailto and tel links have target="_top"
  • Anchor links (#section) have no target attribute
  • Viewport meta tag is set: <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Body and main content use relative widths so the tool reflows on narrow iframes

On the embedding page (Kajabi)

  • Iframe has a unique id
  • Iframe src uses the full GitHub Pages URL with correct case
  • Iframe style includes width: 100%; border: 0;
  • Receiver script is on the same Kajabi page, with id matching the iframe
  • Iframe has a title attribute for screen readers
  • loading="lazy" set if the iframe is below the fold

For Canvas embedding

  • Decide whether linking is better (usually it is)
  • If embedding, set an explicit height value (no auto-sizing in Canvas)
  • Test from a student account if possible, since instructor view differs
